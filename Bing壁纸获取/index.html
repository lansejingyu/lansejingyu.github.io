前言

Bing 搜索引擎每天都会更新一张精美的高清壁纸，这些壁纸质量极高，涵盖自然风光、人文景观、动物植物等各种主题。作为一个博客爱好者，我想把这些壁纸自动收集起来，展示在博客的图库中。

本文将详细介绍如何实现一个完全自动化、零成本、高性能的 Bing 壁纸获取与展示系统。

最终效果

  • ✅ 每天自动获取 Bing 4K 超高清壁纸（3840x2160）
  • ✅ 自动保存到 GitHub 仓库
  • ✅ 通过 CDN 加速访问
  • ✅ 在博客图库中实时展示
  • ✅ 鼠标悬停显示壁纸日期和版权信息
  • ✅ 完全免费，无需服务器

在线预览：博客图库 - Bing Wallpaper 4K

整体架构

这是一个自动化获取 + 远程托管 + 博客展示的三层架构：

技术栈

  • Python - 爬虫脚本
  • GitHub Actions - 定时任务
  • GitHub Repository - 数据存储
  • jsDelivr CDN - 全球加速
  • Hexo + Butterfly 主题 - 博客展示

5.3 关键技术：jsDelivr CDN

为什么使用 jsDelivr？

直接访问 GitHub 原始文件有以下问题：

  • 国内访问速度慢或无法访问
  • 没有 CORS 头，无法跨域请求
  • 没有缓存优化

jsDelivr CDN 的优势：

  • ✅ 全球 CDN 加速，国内访问友好
  • ✅ 自动添加 Access-Control-Allow-Origin: * 响应头
  • ✅ 智能缓存，访问速度快
  • ✅ 完全免费
jsDelivr 在本项目中的使用方式

1. GitHub 原始文件访问方式对比

访问方式 URL 格式 优点 缺点
GitHub Raw https://raw.githubusercontent.com/用户名/仓库名/分支/文件路径 官方直链 国内访问慢/被墙，无 CORS 支持
jsDelivr CDN https://cdn.jsdelivr.net/gh/用户名/仓库名@分支/文件路径 全球加速，支持 CORS 有缓存延迟（通常几分钟）

3. jsDelivr 的版本控制

jsDelivr 支持多种版本指定方式：

方式 URL 示例 说明 适用场景
指定分支 @master 始终获取最新版本 需要实时更新的数据（本项目使用）
指定 Tag @v1.0.0 固定版本，不会变化 稳定的库文件
指定 Commit @a1b2c3d 精确到某次提交 需要特定版本
省略版本 不加 @ 自动使用最新 release 有版本发布的项目

**本项目使用 @master**：

  • 优点：每次 GitHub Actions 更新后，CDN 会自动同步最新数据
  • 缺点：有 1-5 分钟的缓存延迟

4. 缓存刷新策略

jsDelivr 的缓存机制：

  • 默认缓存时间：7 天
  • 自动更新：检测到 GitHub 文件变化后，1-5 分钟内更新
  • 手动刷新：可以通过 jsDelivr 官网手动清除缓存

可能遇到的问题

1. GitHub Actions 没有自动运行

解决方法：

  • 检查仓库设置中是否启用了 Actions
  • 检查 Workflow permissions 是否设置为 Read and write
  • 手动触发一次测试（Actions 页面 → Run workflow）

2. CDN 缓存没有更新

解决方法：

  • jsDelivr 有缓存机制，可能需要等待几分钟
  • 可以在 URL 后添加版本号强制刷新：

3. 图片加载失败

解决方法：

  • 检查 JSON 文件格式是否正确
  • 检查图片 URL 是否有效
  • 查看浏览器控制台的错误信息
